# GH-14: Improve dash start issue format with Background, ACs, and Notes sections

## Spec
- `/issue` currently creates GH issues with an empty body; structured sections (Background, ACs, Notes) make issues self-contained and AI-parseable
- One-liner fast path (`/issue "fix typo"`) must still work — empty sections rather than failure
- `/refine` already reads issue body via `gh issue view --json title,body,comments`; a consistent body structure improves spec generation quality without prompt changes
- Template should be defined in `.dash/config.yaml` (optional) so teams can customize sections; fall back to default if absent
- Applies to both the `/issue` slash command and `_dash_install_commands` copy in `dash.sh`

## Tasks
- [x] Update `/issue` slash command to generate a structured body with Background, ACs, and Notes sections populated from `$ARGUMENTS`
- [x] Support fast path: if `$ARGUMENTS` is a short one-liner, place it in Background and leave ACs/Notes empty
- [x] Add optional `issue_template` key to `.dash/config.yaml` scaffold in `_dash_init`
- [x] Update `_dash_install_commands` in `dash.sh` to install the new `issue.md` content
- [ ] Verify `/refine` spec quality improves with structured body (manual test)
- [x] Update README.md example to show structured issue output
