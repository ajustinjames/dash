# GH-7: Make dash context actually assemble context instead of printing raw instructions

## Spec
- `dash context {cmd} [args]` should resolve and inline the referenced files rather than printing raw command markdown with `$ARGUMENTS` substituted
- Output: config values, relevant history.log lines, current active file contents, matched decision files — all assembled into a ready-to-paste block
- Each command (`refine`, `ask`, `release`) has different context needs; assembly logic per command
- Keep output concise — history.log truncated to post-last-RL if >50 lines (mirrors slash command behaviour)

## Tasks
- [ ] Define context assembly per command: `refine` needs issue body (`gh issue view`), active file titles; `ask` needs history + active files + decisions; `release` needs history since last RL
- [ ] Update `_dash_context` to dispatch per-command assembly instead of raw file substitution
- [ ] Implement `_dash_context_refine {issue}` — fetch GH issue, list active titles
- [ ] Implement `_dash_context_ask {question}` — read history, current active file, matched decisions
- [ ] Implement `_dash_context_release {tag}` — read history since last RL, list closed issues
- [ ] Add fallback: unknown command still prints raw file (current behaviour)
- [ ] Update README `context` entry to reflect assembled output
