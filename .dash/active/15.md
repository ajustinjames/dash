# GH-15: Add /revise command to update specs mid-work

## Spec
- New `/revise` slash command updates an existing `.dash/active/{issue}.md` spec in-place when implementation diverges from the original plan
- Takes `$ARGUMENTS` as optional issue number (detects from branch if omitted) plus user-provided description of what changed
- Regenerates or surgically updates Spec and Tasks sections; preserves Log section and appends a NT entry
- Posts updated spec as a new comment on the GH issue for team visibility
- Parallel to GH-10 (task granularity) and GH-14 (structured issue body) — revised specs should match those conventions
